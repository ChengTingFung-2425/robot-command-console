# 專案記憶 - 專題文件目錄

此目錄包含從主要專案記憶中提取的專題經驗教訓文件。

## 📁 文件結構

| 文件 | 內容 | 行數估計 |
|------|------|---------|
| `phase3_lessons.md` | Phase 3 系列開發經驗 | ~800 行 |
| `cli_batch_lessons.md` | CLI 批次操作經驗 | ~300 行 |
| `tui_llm_lessons.md` | TUI + LLM 整合經驗 | ~400 行 |
| `security_lessons.md` | 安全性相關經驗 | ~300 行 |
| `code_quality_lessons.md` | 代碼品質優化經驗 | ~200 行 |

## 📖 使用方式

### AI 助手
1. 首先讀取主檔案（`docs/PROJECT_MEMORY.md`）獲取概覽
2. 根據需要深入閱讀相關專題文件
3. 任務完成後更新對應的專題文件

### 開發者
1. 快速參考：查看主檔案的 Top 15 經驗
2. 深入學習：閱讀相關專題文件
3. 歷史查詢：查看完整備份（`docs/PROJECT_MEMORY_FULL_BACKUP.md`）

## 🔄 維護指引

### 新增經驗教訓
1. 判斷經驗的重要性和通用性
2. 如果是高頻使用的經驗，加入主檔案的 Top 15
3. 詳細內容加入對應的專題文件
4. 更新主檔案的索引

### 更新現有內容
1. 直接更新對應的專題文件
2. 如果影響 Top 15，同步更新主檔案
3. 保持索引的準確性

## 🎯 設計原則

1. **分層架構**：主檔案精簡，專題文件詳細
2. **快速索引**：主檔案提供清晰的導航
3. **資訊完整**：所有資訊都保留在系統中
4. **易於維護**：模組化設計，便於更新
5. **AI 友善**：主檔案可快速完整讀取
